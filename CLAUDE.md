# makeAiFactory - Claude Code 作業時の指示

## エラー報告の自動チェック（セッション開始時に必ず実行）

このプロジェクトで作業を開始する際、ユーザーへの最初の応答を行う前に、以下を確認すること。

1. `devs/error_reports/` ディレクトリが存在する場合、配下の各 `<report_id>/` サブディレクトリを確認する。
2. `<report_id>/.reviewed` マーカーファイルが存在しないディレクトリ（＝未提示の報告）を探す。
3. 未提示の報告が1件以上あれば:
   - 各 `report.json`（タイトル・メッセージ・システム情報・vram_mode等）と `log_excerpt.txt`
     （存在する場合）を読む。
   - 考えられる原因と、修正が必要そうな箇所・方針を日本語で簡潔にまとめる。
   - ユーザーに「N件の未確認エラー報告があります」として概要を提示し、修正に着手してよいか確認する。
   - 提示した段階で、各 `<report_id>/` に空の `.reviewed` ファイルを作成し、次回以降の
     セッションで同じ報告を再提示しないようにする（実際に修正したかどうかとは無関係に、
     「ユーザーへ提示済み」であることを記録するマーカー）。
4. 未提示の報告が無ければ、このチェックについて言及せず通常の応答を始める。
5. このチェックはユーザーからの最初のメッセージに対する応答の冒頭で行う。

この仕組みは、Discordの通知や自動チェック（タスクスケジューラ）の結果を開発者が見落としても、
次にこのプロジェクトでClaude Codeに何か作業を依頼した時点で必ず報告に気づけるようにするため。

**実際のコード修正は、ユーザーが明示的に許可した後にのみ行うこと。提示と修正可否の確認だけを
自動で行い、無断での修正・コミット・プッシュ・リリースは絶対に行わない。**
